    Ollama JSON Format Specification

    Overview

    This document provides detailed specifications for the JSON request and response formats used when interacting with Ollama models. Ollama supports function calling, multimodal input, and other features. Its JSON format differs from the OpenAI format.

    Core Features: - Support for function calling - Support for multimodal input (images) - Support for thinking process - Use options field to configure model parameters - Response contains detailed execution statistics

    Request Format

    Basic Request Format

    The Ollama request body contains the following main fields:

    • messages: Message array containing conversation history
    • model: Model name (e.g., "qwen3-vl:4b")
    • options: Model configuration options
    • stream: Whether to stream responses
    • tools: Tool definition array (optional)

    Basic Request Example

    {
  "messages": [
    {
      "content": "You are a helpful assistant with multiple tools. When answering user questions, please choose and invoke tools appropriately based on the content of the question, and clearly explain the purpose and results of the tool after invocation. Current time: 2025-12-11 17:25:55",
      "role": "system"
    },
    {
      "content": "hello",
      "role": "user"
    }
  ],
  "model": "qwen3-vl:4b",
  "options": {
    "num_predict": 4096,
    "temperature": 0.7,
    "top_p": 0.95
  },
  "stream": false,
  "tools": [
    {
      "function": {
        "description": "Search the web using keywords to retrieve general information that does not change over time. Supports filtering by country and other conditions to get web content, titles, and link information",
        "name": "search_web2",
        "parameters": {
          "properties": {
            "country": {
              "description": "Country name (must be in English), used to limit search results from a specific country",
              "enum": ["china", "united states", "zimbabwe"],
              "type": "string"
            },
            "query": {
              "description": "Search query string for searching relevant information on the web. Keyword combinations should be concise and clear, avoiding redundant information while ensuring they accurately reflect the core needs of the user's question. Keyword combinations should conform to search engine syntax and logical rules",
              "minLength": 1,
              "type": "string"
            }
          },
          "required": ["query"],
          "type": "object"
        }
      },
      "type": "function"
    }
  ]
}

    Request Format with Tool Calls

    When the model needs to call a tool, the request body will include the assistant's tool call message and the tool's execution result:

    {
  "messages": [
    {
      "content": "You are a helpful assistant with multiple tools. When answering user questions, please choose and invoke tools appropriately based on the content of the question, and clearly explain the purpose and results of the tool after invocation. Current time: 2025-12-11 17:32:52",
      "role": "system"
    },
    {
      "content": "tell me one news from San Francisco today.",
      "role": "user"
    },
    {
      "content": "",
      "role": "assistant",
      "tool_calls": [
        {
          "function": {
            "arguments": {
              "country": "united states",
              "query": "news San Francisco today"
            },
            "name": "search_web2"
          },
          "id": "call_qj2w6nig"
        }
      ]
    },
    {
      "content": "{\"msg\":\"\",\"responses\":[{\"score\":0.55072874,\"snippet\":\"Exclusive Bay Area and San Francisco breaking news, sports, tech, and food and wine coverage, plus enhanced coverage of Giants, 49ers and Warriors.\",\"title\":\"San Francisco Chronicle - Bay Area news, sports, tech, food\",\"url\":\"https://www.sfchronicle.com/\"},...],\"status\":\"success\"}",
      "role": "tool",
      "tool_call_id": "call_qj2w6nig",
      "tool_name": "search_web2"
    }
  ],
  "model": "qwen3-vl:4b",
  "options": {
    "num_predict": 4096,
    "temperature": 0.7,
    "top_p": 0.95
  },
  "stream": false,
  "tools": [...]
}

    Multimodal Request Format

    Ollama supports including images (Base64 encoded) in user messages:

    {
  "messages": [
    {
      "content": "You are a helpful assistant with multiple tools. When answering user questions, please choose and invoke tools appropriately based on the content of the question, and clearly explain the purpose and results of the tool after invocation. Current time: 2025-12-11 18:02:05",
      "role": "system"
    },
    {
      "content": "What is this",
      "images": ["iVBORw0KGgo_THIS_IS_BASE64_EXAMPLE_JRU5ErkJggg=="],
      "role": "user"
    }
  ],
  "model": "qwen3-vl:4b",
  "options": {
    "num_predict": 4096,
    "temperature": 0.7,
    "top_p": 0.95
  },
  "stream": false,
  "tools": []
}

    Response Format

    Basic Response Format

    The Ollama response contains the following fields:

    • code: HTTP status code
    • raw_data: Raw response data
    • role: Message role
    • tool_calls: Tool call array (if present)

    {
  "code": 200,
  "raw_data": {
    "created_at": "2025-12-11T09:48:49.196890089Z",
    "done": true,
    "done_reason": "stop",
    "eval_count": 342,
    "eval_duration": 1638869426,
    "load_duration": 99627223,
    "message": {
      "content": "",
      "role": "assistant",
      "thinking": "Okay, the user is asking for one news article from San Francisco today. Let me see how to approach this.\n\nFirst, I need to check if there's a tool that can search the web for news. The available tool is search_web2, which takes a query and a country. The country parameter is required to be an English country name from the enum list. But San Francisco is a city in the US, so the country should be \"united states\".\n\nWait, the user specified \"San Francisco today\", so the query should be something like \"news in San Francisco today\". The function's description says it's for getting general info that doesn't change over time, but news might be time-sensitive. However, since the user wants today's news, maybe the tool can still handle it.\n\nWait, the tool's parameters include country, which is a required field. The country here is United States. So I need to set country to \"united states\" and query to \"news in San Francisco today\".\n\nLet me check the parameters again. The query is a string, so \"news in San Francisco today\" makes sense. The country should be \"united states\" from the enum list. Let me confirm the country names. The enum includes \"united states\" as one of the options. Yes, it's there.\n\nSo the tool call would be search_web2 with country \"united states\" and query \"news in San Francisco today\". That should fetch the latest news articles from San Francisco today.",
      "tool_calls": [
        {
          "function": {
            "arguments": {
              "country": "united states",
              "query": "news in San Francisco today"
            },
            "index": 0,
            "name": "search_web2"
          },
          "id": "call_26219uki"
        }
      ]
    },
    "model": "qwen3-vl:4b",
    "prompt_eval_count": 1075,
    "prompt_eval_duration": 89058402,
    "total_duration": 1978183031
  },
  "role": "assistant",
  "tool_calls": [
    {
      "function": {
        "arguments": "{\"country\":\"united states\",\"query\":\"news in San Francisco today\"}",
        "name": "search_web2"
      },
      "id": "call_26219uki",
      "type": "function"

}

    Key Field Descriptions

    Request Fields

    • messages: Message array supporting system, user, assistant, and tool roles
    • model: Model identifier in the format "model_name:version"
    • options: Model configuration
    • num_predict: Maximum number of tokens to generate
    • temperature: Temperature parameter (0.0-1.0)
    • top_p: Top-p sampling parameter
    • stream: Whether to stream responses (true/false)
    • tools: Tool definition array, each tool contains function and type fields

    Response Fields

    • code: HTTP status code
    • raw_data: Contains complete response information
    • created_at: Creation timestamp
    • done: Whether completed
    • done_reason: Completion reason
    • eval_count: Number of evaluation tokens
    • eval_duration: Evaluation duration (nanoseconds)
    • load_duration: Load duration (nanoseconds)
    • message: Message content, may contain thinking field (thinking process)
    • prompt_eval_count: Number of prompt evaluation tokens
    • prompt_eval_duration: Prompt evaluation duration (nanoseconds)
    • total_duration: Total duration (nanoseconds)
    • role: Message role
    • tool_calls: Tool call array (if the model decides to call tools)

    Tool Calling Workflow

    1. Initial Request: Send user message and tool definitions
    2. Tool Call Response: Model returns tool_calls containing tools to call and parameters
    3. Tool Execution: Execute tools and obtain results
    4. Subsequent Request: Add tool execution results as tool role messages to message history
    5. Final Response: Model generates final answer based on tool results

    Notes

    1. Thinking Process: The thinking field in Ollama responses contains the model's reasoning process, which is useful for debugging
    2. Parameter Format: Tool call arguments in responses may be strings (JSON format) or objects
    3. Multimodal Support: Images need to be Base64 encoded and placed in the images array
    4. Time Statistics: Responses contain detailed performance statistics that can be used for monitoring and optimization
    5. Tool Result Format: Tool execution results need to be placed as JSON strings in the content field
