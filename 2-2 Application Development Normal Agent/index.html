    Normal Agent Development Example

    Overview

    The normal agent demonstrates how to use HADK's sequential workflow with context compression to implement a three-stage question-answering system. This example implements an agent that compresses conversation context, generates responses using LLM with tool support, and then polishes the final answer.

    Workflow Diagram:

    Input → ce_node (Context Compression) → generate_node (Generate Response) → polish_node (Polish Response) → Output

    Core Features: - Context compression: Automatically compresses conversation history when it exceeds the limit using SUMMARIZING strategy - Tool support: Generate node can use tools (e.g., search_web2) to gather information - Response polishing: Final polish node refines the generated response for better quality - Sequential workflow: Linear flow through three specialized nodes

    Development Steps

    1. Register Tools

    Same as a single node agent, you need to register the required tools first. It's recommended to wrap tool registration in separate functions:

    #include <common_tools/tools.h>
#include <web_search.h>

std::string init_tools() {
    // Register local tools
    common_tools::tools::add_function_call(search_web_tool,
        R"({"type":"function","function":{"name":"search_web2","description":"使用关键词搜索网页，用于获取通用的，不会随着时间变化而改变的信息，支持按国家等过滤条件获取网页内容、标题和链接信息","parameters":{"type":"object","properties":{"query":{"type":"string","description":"搜索查询字符串，用于在网络上搜索相关信息。关键词组合应简洁明了，避免冗余信息，同时确保能够准确反映用户问题的核心需求。关键词组合应符合搜索引擎的语法和逻辑规则","minLength":1},"country":{"type":"string","description":"国家名称(必须是英文)，用于限定搜索结果来自的国家","enum":["afghanistan","albania",...]}},"required":["query"]}}})"
    );
    return common_tools::tools::get_all_tools_json();
}

std::string close_tools() {
    try {
        common_tools::tools::shutdown_all_servers();
    } catch (...) {
        // Ignore exceptions during shutdown
    }
    return "Tools closed successfully";
}

    2. Configure Context Compression Node (CE Node)

    The CE (Context Compression) node is responsible for compressing conversation history when it exceeds the limit. It uses the SUMMARIZING strategy:

    #include <ce_node/ce_node.h>

ce_node::ce_node_settings s_ce;

// Use SUMMARIZING strategy (alternative: TRIMMING)
s_ce.strategy = ce_node::ContextStrategy::SUMMARIZING;
s_ce.context_limit = 3;              // Context limit to trigger compression
s_ce.keep_last_n_turns = 1;         // Keep last n turns of original messages
s_ce.tool_trim_limit = 600;         // Tool results are not important in history, so the first 600 characters of tool results will be kept
s_ce.summarizer_model = "gpt-4o-mini";
s_ce.summarizer_max_tokens = 400;

const auto ce_node = std::make_shared<ce_node::CeNode<std::string, std::string>>(s_ce);

    Configuration Parameters: - strategy: Compression strategy (SUMMARIZING or TRIMMING) - context_limit: Number of conversation turns before triggering compression - keep_last_n_turns: Number of recent turns to keep in original form - tool_trim_limit: Maximum characters to keep from tool results - summarizer_model: Model used for summarization - summarizer_max_tokens: Maximum tokens for summarization

    Alternative Configuration (TRIMMING strategy):

    s_ce.strategy = ce_node::ContextStrategy::TRIMMING;
s_ce.max_turns = 3;  // Max turns to keep in history

    3. Configure Routing for CE Node

    Use the route function to set the routing value for the CE node output:

    route(ce_node, [&](const std::string&, const std::string& output) -> std::optional<std::string> {
    // Setting routing value 'generate'
    return "generate";
});

    Parameter Description: - First parameter: Source node (ce_node) - Second parameter: Route function that receives input and output, returns route name (std::optional<std::string>)

    4. Configure Generate Node

    The generate node uses LLM to generate responses and supports tool calling:

    #include <chat_node/chat_node.h>
#include <common_tools/tools.h>

chat_node::chat_node_settings s_generate;
s_generate.model = "gpt-4o-mini";   // Model to use
s_generate.temperature = 0.7;        // Temperature parameter
s_generate.top_p = 0.95;             // Top-p sampling parameter
s_generate.max_tokens = 4096;        // Maximum generated tokens
s_generate.tool_choice = "auto";     // Tool selection strategy: auto/none/required
s_generate.tools_json = common_tools::tools::get_all_tools_json();  // Get all registered tools

const auto generate_node = std::make_shared<chat_node::ChatNode<std::string, std::string>>(s_generate);

    5. Configure Routing for Generate Node

    Set the routing value for the generate node output:

    route(generate_node, [&](const std::string&, const std::string& output) -> std::optional<std::string> {
    // Setting routing value 'polish'
    return "polish";
});

    6. Configure Polish Node

    The polish node refines the generated response without tool support:

    chat_node::chat_node_settings s_polish;
s_polish.model = "gpt-4o-mini";
s_polish.temperature = 0.7;
s_polish.top_p = 0.95;
s_polish.max_tokens = 4096;
s_polish.tool_choice = "none";  // No tool support for polishing

const auto polish_node = std::make_shared<chat_node::ChatNode<std::string, std::string>>(s_polish);

    7. Configure Polish Node Preprocessor

    The polish node uses a preprocessor to modify the input prompt:

    polish_node->setPreprocessor([](const std::string& in) -> std::string {
    // Modify the last message of the input of current node
    nlohmann::json inJson = nlohmann::json::parse(in);
    std::string _content = inJson.back()["content"].get<std::string>();

    std::string prompt = R"(
    ### CONTEXT
    polish the following content:
    Content: )" + _content + R"(
    ## YOUR ANSWER:
    Provide a comprehensive answer using the content.)";

    inJson.back()["content"] = prompt;
    return inJson.dump();
});

    8. Connect Nodes

    Use the chain function to establish sequential connections between nodes:

    // CE node to generate node
nodeflow::chain(ce_node, generate_node, "generate");

// Generate node to polish node
nodeflow::chain(generate_node, polish_node, "polish");

    Parameter Description: - First parameter: Source node - Second parameter: Target node - Third parameter: Route name (must match the routing value set by route function)

    9. Create Workflow and Execute

    // Create workflow
auto koba_flow = std::make_shared<nodeflow::Flow>();

// Set start node
koba_flow->start(ce_node);

// Execute workflow
auto result = koba_flow->runWithInput<std::string, std::string>(message);

    Execution Flow: 1. Input message enters ce_node 2. ce_node compresses context if it exceeds the limit 3. Compressed context flows to generate_node 4. generate_node generates response, can use tools if needed 5. Generated response flows to polish_node 6. polish_node refines the response 7. Return final polished answer

std::string koba_agent(const std::string& message) {
    std::cout << "Running agent..." << std::endl;

    // Configure CE node
    ce_node::ce_node_settings s_ce;
    s_ce.strategy = ce_node::ContextStrategy::SUMMARIZING;
    s_ce.context_limit = 3;              // Context limit to trigger compression
    s_ce.keep_last_n_turns = 1;          // Keep last n turns of original messages
    s_ce.tool_trim_limit = 600;           // Tool results are not important in history, so the first 600 characters of tool results will be kept
    s_ce.summarizer_model = "gpt-4o-mini";
    s_ce.summarizer_max_tokens = 400;

    const auto ce_node = std::make_shared<ce_node::CeNode<std::string, std::string>>(s_ce);

    // Configure routing for CE node
    route(ce_node, [&](const std::string&, const std::string& output) -> std::optional<std::string> {
        // Setting routing value 'generate'
        return "generate";
    });

    // Configure generate node
    chat_node::chat_node_settings s_generate;
    s_generate.model = "gpt-4o-mini";     // Model to use
    s_generate.temperature = 0.7;         // Temperature
    s_generate.top_p = 0.95;             // Top-p
    s_generate.max_tokens = 4096;         // Max tokens
    s_generate.tool_choice = "auto";      // Tool choice
    s_generate.tools_json = common_tools::tools::get_all_tools_json(); // Tools json

    const auto generate_node = std::make_shared<chat_node::ChatNode<std::string, std::string>>(s_generate);

    // Configure routing for generate node
    route(generate_node, [&](const std::string&, const std::string& output) -> std::optional<std::string> {
        // Setting routing value 'polish'
        return "polish";
    });

    // Configure polish node
    chat_node::chat_node_settings s_polish;
    s_polish.model = "gpt-4o-mini";
    s_polish.temperature = 0.7;
    s_polish.top_p = 0.95;
    s_polish.max_tokens = 4096;
    s_polish.tool_choice = "none";

    const auto polish_node = std::make_shared<chat_node::ChatNode<std::string, std::string>>(s_polish);

    // Configure polish node preprocessor
    polish_node->setPreprocessor([](const std::string& in) -> std::string {
        // Modify the last message of the input of current node
        nlohmann::json inJson = nlohmann::json::parse(in);
        std::string _content = inJson.back()["content"].get<std::string>();
        std::string prompt = R"(
        ### CONTEXT
        polish the following content:
        Content: )" + _content + R"(
        ## YOUR ANSWER:
        Provide a comprehensive answer using the content.)";

        inJson.back()["content"] = prompt;
        return inJson.dump();
    });

    // Chain the nodes to a flow
    nodeflow::chain(ce_node, generate_node, "generate");
    nodeflow::chain(generate_node, polish_node, "polish");

    // Create flow
    auto koba_flow = std::make_shared<nodeflow::Flow>();
    koba_flow->start(ce_node);  // Start flow with CE node

    // Run flow with message
    auto result = koba_flow->runWithInput<std::string, std::string>(message);

    return result;  // Return response from flow
}

int main() {
    // Initialize tools
    init_tools();

    // Build input message (OpenAI Chat Completion format)
    nlohmann::json inputJson = nlohmann::json::array();
    inputJson.push_back({
        {"role", "user"},
        {"content", "What is the capital of France?"}
    });

    // Call agent and get response
    std::string response = koba_agent(inputJson.dump());

    // Output result
    std::cout << response << std::endl;

    // Close tools
    close_tools();

    return 0;
}

    Key Points

    1. Context Compression: The CE node automatically compresses conversation history when it exceeds the limit, using either SUMMARIZING or TRIMMING strategy
    2. Sequential Workflow: Nodes are connected in a linear sequence using the chain function
    3. Routing Values: Use the route function to set routing values for node outputs, which are used by chain to determine connections
    4. Tool Support: The generate node supports tool calling, allowing it to use tools like search_web2 to gather information
    5. Response Polishing: The polish node refines the generated response without tool support, ensuring high-quality final output
    6. Preprocessor Usage: The polish node uses a preprocessor to modify the input prompt, instructing the LLM to polish the content
