    OpenAI JSON 格式说明

    Overview

    本文档详细说明了与 OpenAI 兼容 API 交互时使用的 JSON 请求和响应格式。OpenAI 格式是业界标准，被许多 LLM 服务提供商采用。

    核心特性： - 标准化的消息格式 - 支持工具调用（Function Calling） - 使用 max_tokens 控制输出长度 - 支持 tool_choice 参数控制工具使用策略 - 响应格式符合 OpenAI API 规范

    请求格式

    基础请求格式

    OpenAI 的请求体包含以下主要字段：

    • messages: 消息数组，包含对话历史
    • model: 模型名称（如 "gpt-4o-mini"）
    • max_tokens: 最大生成 token 数
    • temperature: 温度参数
    • top_p: Top-p 采样参数
    • stream: 是否流式返回
    • tools: 工具定义数组（可选）
    • tool_choice: 工具选择策略（可选）
    • chat_template_kwargs: 聊天模板参数（可选）

    基础请求示例

    {
  "chat_template_kwargs": {
    "enable_thinking": false
  },
  "max_tokens": 4096,
  "messages": [
    {
      "content": "You are a helpful assistant with multiple tools. When answering user questions, please choose and invoke tools appropriately based on the content of the question, and clearly explain the purpose and results of the tool after invocation. Current time: 2025-12-11 17:36:42",
      "role": "system"
    },
    {
      "content": "tell me one news from San Francisco today.",
      "role": "user"
    }
  ],
  "model": "gpt-4o-mini",
  "stream": false,
  "temperature": 0.7,
  "tool_choice": "auto",
  "tools": [
    {
      "function": {
        "description": "使用关键词搜索网页，用于获取通用的，不会随着时间变化而改变的信息，支持按国家等过滤条件获取网页内容、标题和链接信息",
        "name": "search_web2",
        "parameters": {
          "properties": {
            "country": {
              "description": "国家名称(必须是英文)，用于限定搜索结果来自的国家",
              "enum": ["china", "united states", "zimbabwe"],
              "type": "string"
            },
            "query": {
              "description": "搜索查询字符串，用于在网络上搜索相关信息。关键词组合应简洁明了，避免冗余信息，同时确保能够准确反映用户问题的核心需求。关键词组合应符合搜索引擎的语法和逻辑规则",
              "minLength": 1,
              "type": "string"
            }
          },
          "required": ["query"],
          "type": "object"
        }
      },
      "type": "function"
    }
  ],
  "top_p": 0.95
}

    带工具调用的请求格式

    当模型需要调用工具时，请求体中会包含 assistant 的工具调用消息和 tool 的工具执行结果：

  "chat_template_kwargs": {
    "enable_thinking": false
  },
  "max_tokens": 4096,
  "messages": [

      "role": "system"
    },
    {
      "content": "tell me one news from San Francisco today.",
      "role": "user"
    },
    {
      "content": "",
      "role": "assistant",
      "tool_calls": [
        {
          "function": {
            "arguments": "{\"country\":\"united states\",\"query\":\"San Francisco news December 11 2025\"}",
            "name": "search_web2"
          },
          "id": "call_hMffYLcob2f6Q3FIxllY8xuy",
          "type": "function"
        }
      ]
    },
    {
      "content": "{\"msg\":\"\",\"responses\":[{\"score\":0.8020137,\"snippet\":\"Thursday Dec 11, 2025 9:00 AM - 12:00 PM PST. Location. Center for Architecture + Design 140 Sutter Street San Francisco, CA 94104. Fees/Admission. $10.00\",\"title\":\"BuildSF Summit: The Next Chapter for Downtown ...\",\"url\":\"https://business.sfchamber.com/events/details/buildsf-summit-the-next-chapter-for-downtown-san-francisco-12899\"},...],\"status\":\"success\"}",

    响应格式

    基础响应格式

    OpenAI 的响应包含以下字段：

    • code: HTTP 状态码
    • raw_data: 原始响应数据（符合 OpenAI API 规范）
    • role: 消息角色
    • tool_calls: 工具调用数组（如果存在）

    关键字段说明

    请求字段

    • messages: 消息数组，支持 system、user、assistant、tool 角色
    • model: 模型标识符
    • max_tokens: 最大生成 token 数（替代 Ollama 的 options.num_predict）
    • temperature: 温度参数（0.0-2.0）
    • top_p: Top-p 采样参数（0.0-1.0）
    • stream: 是否流式返回（true/false）
    • tools: 工具定义数组，每个工具包含 function 和 type 字段
    • tool_choice: 工具选择策略
    • "auto": 自动决定是否调用工具
    • "none": 不调用工具
    • "required": 必须调用工具
    • {"type": "function", "function": {"name": "tool_name"}}: 指定调用特定工具
    • chat_template_kwargs: 聊天模板参数，如 enable_thinking 控制是否启用思考过程

    响应字段

    • code: HTTP 状态码
    • raw_data: 符合 OpenAI API 规范的响应数据
    • choices: 选择数组，包含生成的响应
      • finish_reason: 完成原因（"stop"、"tool_calls"、"length" 等）
      • message: 消息内容
      • content: 文本内容（可能为 null）
      • role: 消息角色
      • tool_calls: 工具调用数组
      • annotations: 注释数组
      • refusal: 拒绝内容（如果有）
    • created: 创建时间戳（Unix 时间戳）
    • id: 响应 ID
    • model: 使用的模型
    • object: 对象类型（"chat.completion"）
    • usage: Token 使用统计
      • prompt_tokens: 提示 token 数
      • completion_tokens: 完成 token 数
      • total_tokens: 总 token 数
      • prompt_tokens_details: 提示 token 详细信息
      • completion_tokens_details: 完成 token 详细信息
    • role: 消息角色
    • tool_calls: 工具调用数组（如果模型决定调用工具）

    工具调用流程

    1. 初始请求: 发送用户消息和工具定义，设置 tool_choice 为 "auto"
    2. 工具调用响应: 模型返回 tool_calls，包含要调用的工具和参数
    3. 工具执行: 执行工具并获取结果
    4. 后续请求: 将工具执行结果作为 tool 角色的消息添加到消息历史中
    5. 最终响应: 模型基于工具结果生成最终回答

    与 Ollama 格式的主要区别

    1. 参数配置: OpenAI 使用顶层字段（max_tokens、temperature），Ollama 使用 options 对象
    2. 工具选择: OpenAI 支持 tool_choice 参数，提供更灵活的工具调用控制
    3. 响应结构: OpenAI 响应符合标准 API 规范，包含 choices 数组和 usage 统计
    4. 思考过程: OpenAI 通过 chat_template_kwargs.enable_thinking 控制，而 Ollama 在响应中直接包含 thinking 字段
    5. 时间统计: OpenAI 提供 token 使用统计，Ollama 提供详细的性能时间统计
    6. 工具参数格式: OpenAI 的工具调用 arguments 始终为 JSON 字符串

    注意事项

    1. 工具参数: 工具调用的 arguments 必须是 JSON 字符串格式
    2. finish_reason: 当模型决定调用工具时，finish_reason 为 "tool_calls"
    3. content 字段: 当模型调用工具时，content 可能为 null
    4. tool_choice: 合理使用 tool_choice 可以控制模型的行为，避免不必要的工具调用
    5. Token 统计: usage 字段提供了详细的 token 使用情况，可用于成本计算和优化
